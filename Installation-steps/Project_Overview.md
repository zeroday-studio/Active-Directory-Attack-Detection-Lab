# 📘 Project Overview — Active Directory Attack & Detection Lab

## 📌 Introduction

This project demonstrates a real-world Active Directory (AD) environment integrated with a SIEM solution (Splunk) to simulate both cyber attacks and detection mechanisms.

The lab replicates an enterprise network where users, systems, and logs are centrally managed and monitored. It combines offensive security techniques, threat simulation using MITRE ATT&CK, and defensive monitoring using SIEM.

---

## 🎯 Objective

The main objective of this project is to:

- Build an Active Directory environment using Windows Server 2022  
- Create domain users and organizational units  
- Simulate real-world attacks such as RDP brute-force  
- Perform threat emulation using Atomic Red Team (MITRE ATT&CK)  
- Monitor and detect attack activity using Splunk SIEM  
- Understand how logs are generated and analyzed in a SOC environment  

---

## 🏗️ Lab Environment Overview

The lab consists of multiple machines connected in the same network:

- **Domain Controller (Windows Server 2022)**  
  - Configured as Active Directory  
  - Domain: `rocky.local`  

- **Windows 10 (Target Machine)**  
  - Joined to the domain  
  - Used as victim machine for attack simulation  

- **Kali Linux (Attacker Machine)**  
  - Used to perform reconnaissance and brute-force attacks  

- **Splunk Server (Ubuntu Server)**  
  - Used as SIEM for log collection and analysis  

---

## ⚙️ Active Directory Configuration

- Installed and configured **Windows Server 2022**  
- Promoted server to **Domain Controller**  
- Domain name configured as: `rocky.local`  

### 👥 User & OU Setup

Using **Active Directory Users and Computers**:

- Created Organizational Units (OUs):
  - HR  
  - IT  

- Created users:
  - `jsmith` → HR Department  
  - `asingh` → IT Department

## 🖥️ Windows 10 Configuration

- Installed Windows 10 and joined it to the domain `rocky.local`  
- Changed from workgroup to domain using system settings  
- Provided administrative credentials to join domain  
- Restarted system to apply changes  

### 🔐 Remote Access

- Enabled **RDP (Remote Desktop Protocol)** on Windows 10  
- This created an entry point for remote login attacks  

---

## 📊 Splunk SIEM Setup

- Installed **Ubuntu Server** for Splunk deployment  
- Used VirtualBox shared folder to transfer Splunk package  
- Installed Splunk Enterprise (free trial) using CLI  

### 🔧 Configuration

- Accessed Splunk Web UI:  
  `http://192.168.10.10:8000`  

- Created custom index for log storage  
- Configured receiving port **9997** for log ingestion  

---

## 🔄 Log Forwarding Configuration

- Installed **Splunk Universal Forwarder** on:
  - Domain Controller  
  - Windows 10  

- Configured forwarder to send logs to:
  - Splunk Server (192.168.10.10:9997)

- Configured `inputs.conf` to forward:
  - Windows Security Logs

## 🧾 Sysmon Integration

- Installed **Sysmon (sysmon64)** on both systems  
- Used configuration file: `sysmonconfig.xml`  

👉 Purpose:
- Capture detailed system activity such as:
  - Process creation  
  - Network connections  
  - Suspicious behavior

## ⚔️ Atomic Red Team (MITRE ATT&CK Simulation)

- Installed **Atomic Red Team** on Windows 10 (Target Machine)  
- Executed multiple MITRE ATT&CK techniques to simulate attacker behavior  

### 🔍 Techniques Simulated:

- **T1136.001 — Create Local Account**  
- **T1059.001 — PowerShell Execution**  
- Additional MITRE techniques  

### 🎯 Purpose:

- Simulate real-world attack techniques  
- Generate realistic logs  
- Validate detection capabilities  

---

## 📊 Detection of Atomic Attacks

- Logs generated by Atomic Red Team were captured using **Sysmon**  
- Logs were forwarded to **Splunk SIEM**  
- Observed:
  - Suspicious PowerShell execution  
  - Account creation events  
  - Process activity  

---

## 💣 Attack Simulation (RDP Brute Force)

### 1️⃣ Reconnaissance
- Performed network scanning using **Nmap**  
- Identified open services  
- Found **RDP (Port 3389)** enabled  

### 2️⃣ Credential Attack
- Used **Hydra** to perform RDP brute-force attack  
- Targeted domain users:
  - `jsmith` (HR)  
  - `asingh` (IT)  

- Successfully compromised **user `jsmith`**  
- Gained access to Windows 10 system via RDP  

---

## 📊 Detection & Monitoring

- Splunk detected:
  - Multiple failed login attempts
      - log_ID = 4625
  - Successful login after brute-force
      - log_ID = 4624  
- Demonstrated visibility of attack behavior in SIEM
